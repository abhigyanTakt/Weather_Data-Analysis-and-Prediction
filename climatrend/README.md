# ClimaTrend - Weather Analysis and Forecasting System

ClimaTrend is a portfolio-quality meteorological data science project and interactive web application that fetches historical weather station logs, processes the data, generates rich visualizations, and constructs predictive time-series models. 

It includes a premium, responsive **Streamlit Dashboard** that allows users to interactively explore weather statistics, spatial heatmaps/contours, and time-series forecasts alongside natural language weather summaries generated by the **Deepseek v4 Pro** model.

## Tech Stack & Data Sources

### Data Sources
*   **Open-Meteo Archive API**: Free historical weather data back to 1940. No API key required.
*   **Open-Meteo Geocoding API**: Resolves city queries to precise latitude/longitude coordinate offsets.

### Core Libraries
*   `pandas`, `numpy`: Data manipulation, filtering, and aggregation.
*   `meteostat`: Historical station-level parsing.
*   `statsmodels`: Autocorrelation (ACF/PACF) and multi-seasonal additive decomposition.
*   `pmdarima`: Stepwise ARIMA/SARIMA hyperparameter auto-tuning.
*   `prophet`: Multi-seasonal additive forecasting curve fitting.
*   `scikit-learn`: Linear and Random Forest recursive forecasting models.
*   `matplotlib`, `seaborn`, `plotly`: Static and web-interactive data visualizations.
*   `geopandas`, `folium`: Geographic map rendering and HeatMap overlays.
*   `openai`: Handles completions using the Nvidia Deepseek v4 API endpoint.

## Setup Instructions

1.  **Clone / Copy the directory** to your local machine.
2.  **Install dependencies** using `pip`:
    ```bash
    pip install -r requirements.txt
    ```
3.  *(Optional)* Set your Nvidia API key as an environment variable or plug it directly into the dashboard sidebar.
    ```bash
    export DEEPSEEK_API_KEY="your-nvapi-key"
    ```

## Running the Applications

### 1. The Streamlit Web App
To launch the interactive dashboard, run:
```bash
streamlit run app.py
```
This launches a browser session on `http://localhost:8501` featuring:
*   **Overview & Data**: Key performance metrics, aggregations, and downloads.
*   **Exploratory Analysis**: Charts covering monthly distributions, correlations, wind rose polar histograms, calendar grids, and seasonal cycles.
*   **Spatial Analysis**: Global Leaflet heatmaps and temperature gradients.
*   **Forecasting Engine**: Live comparison of ARIMA, Prophet, and Machine Learning models evaluated on RMSE/MAE/MAPE.
*   **AI Climate Analysis**: Expert meteorological reports generated by Deepseek v4.

### 2. The Jupyter Notebook
To run the step-by-step analysis in your notebook server:

```bash
jupyter notebook notebooks/analysis.ipynb
```

## Methodology

### 1. Feature Engineering & Meteorological Aggregations
*   **Trigonometric Wind Vectors**: Wind directions are aggregated using sine and cosine components before reconstructing average degrees (`arctan2`) to prevent cardinal wrap-around errors (e.g. averaging $350^\circ$ and $10^\circ$ as $180^\circ$ instead of $0^\circ$).
*   **Apparent Heat Index & Wind Chill**: Secondary feels-like metrics are computed programmatically using Rothfusz and standard North American meteorological regressions based on humidity/temperature/wind combinations.
*   **Outlier Capping**: Z-score analysis caps anomalous values at $3\sigma$ bounds, avoiding data discontinuities.

### 2. Time-Series Forecasting
*   **Prophet**: Models long-term trend lines alongside weekly and yearly periodic Fourier components. Extremely robust to noise and missing observations.
*   **SARIMAX**: Fits classical statistical coefficients. Evaluated using a stepwise search over auto-regressive (p), differencing (d), and moving average (q) orders.
*   **ML Lags (Recursive)**: Maps the time-series as a regression task. Lags ($1, 2, 7, 30, 365$ days) and rolling averages ($7, 30$ days) are computed recursively step-by-step to prevent information leakage during forecasting.
